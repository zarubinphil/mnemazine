**Mnemazine** — открытая персональная система памяти, названная в честь **Мнемозины**, греческой богини памяти и матери муз.

Идея простая. Большинство людей копят заметки, скриншоты и сохранённые ссылки — и больше к ним не возвращаются. Куча — это не память. Настоящая память появляется, когда ты берёшь сырое, понимаешь его, проверяешь, связываешь с тем, что уже знаешь, и записываешь только то, что стоит сохранить. Mnemazine делает эту часть за тебя прямо в момент сохранения.

У приёма есть имя — **синтез на записи** (synthesis on write), его популяризировал [Андрей Карпаты](https://karpathy.ai/): не сваливай в надежде прочитать потом, а отжимай суть прямо при захвате — чтобы заметка была полезной уже при следующем открытии.

На практике Mnemazine берёт скриншоты, PDF, веб-страницы, YouTube-видео, заметки, гайды и GitHub-репозитории и превращает их в чистую, совместимую с [Obsidian](https://obsidian.md/) базу знаний. Сначала извлекает текст локально, хранит хэш источника — чтобы ты всегда знал, откуда факт, — кладёт в vault только готовые заметки, связывает их в граф и не пускает сырой OCR и черновой мусор внутрь.

> Память, не свалка.

## Что это

Mnemazine — локальная система переработки входящих материалов в долговечные знания. Сырьё на входе — готовое знание на выходе.

Она не кладёт в vault сырой OCR, случайные имена файлов, скриншоты без контекста и непроверяемые копипасты. Вместо этого создаёт финальные заметки:

- понятный заголовок;
- ссылки на источники и статус проверки;
- атомизацию по темам, когда один источник содержит много идей;
- готовые к переиспользованию описания skills, инструкции агентам, заметки по реализации и действия для проектов;
- Graphify-карты для семантической навигации;
- еженедельные HTML-брифинги с локальным статусом: `read`, `work on it`, `forget`.
- визуальные отчёты после прохода: кластеры, малые атомы, учёт дублей и топ-20 рекомендуемых действий.

Цель проста: будущий ты не должен перечитывать двадцать скриншотов, целый гайд или сырой транскрипт. Будущий ты открывает одну хорошую заметку и сразу понимает, что это за знание, почему оно важно, как его применить и чем оно подтверждено.

## Почему это экономит токены

Mnemazine экономит токены, вынося работу из повторяющегося LLM-контекста в долговечную локальную структуру: парсить локально, агрессивно кэшировать, хранить очищенные атомы, извлекать узко.

<p align="center">
  <img src="docs/assets/hero/mnemazine-token.png" width="760" alt="Экономика токенов: парсинг локально, hash-кэш, хранение атомов, узкое извлечение">
</p>

Типичная экономия складывается из:

- **Сначала локальное извлечение:** Apple Vision OCR, парсинг PDF, транскрипция, хэширование и перепись файлов выполняются локально, когда это возможно.
- **Hash-кэш:** повторные файлы обнаруживаются до обработки LLM. Дубликат стоит ноль модельных токенов.
- **Атомизация:** один длинный гайд может стать двадцатью сфокусированными заметками, и будущие промпты подтягивают только нужный атом.
- **Graphify-контекст:** агент может запросить граф вместо того, чтобы вываливать в контекст весь vault.
- **Только финальные заметки:** сырой OCR и шумные транскрипты остаются вне vault. Vault хранит сжатое, проверенное, человекочитаемое знание.
- **Еженедельные брифинги:** система показывает, что изменилось и что заслуживает действия, чтобы пользователь не просил модель заново переоткрывать прошедшую неделю.

В реальных сценариях это часто превращает огромные кучи источников в компактные переиспользуемые заметки. Точная экономия зависит от объёма источника, но рабочий принцип надёжен: парсить локально, агрессивно кэшировать, хранить очищенные атомы, извлекать узко.

## Установка

Склонируйте проект в единственную нужную папку:

```bash
git clone https://github.com/zarubinphil/Mnemazine.git "$HOME/Desktop/Mnemazine"
cd "$HOME/Desktop/Mnemazine"
bash setup.sh        # пошагово, с вопросами (куда класть inbox, опц. Telegram-бот)
# или: bash install.sh   # без вопросов, только каркас
```

`setup.sh` проводит новое устройство по этапам: проверяет окружение, чётко
говорит что доустановить если чего-то нет, спрашивает куда класть `inbox/`
(в репозитории или на рабочий стол), может задеплоить Telegram-бота на VPS.
Прогон вхолостую без изменений: `MNEMAZINE_SETUP_DRYRUN=1 bash setup.sh`.

После установки откройте эту папку как Obsidian-vault:

```text
~/Desktop/Mnemazine/vault
```

По умолчанию всё лежит под `~/Desktop/Mnemazine`:

- `inbox/` — сырые входящие;
- `vault/` — готовое знание;
- `reports/` — еженедельные HTML-брифинги;
- `.mnemazine/` — кэши, бинарники и локальное состояние;
- `skills/`, `agents/`, `workflows/`, `scripts/` — агентная система.

## Требования

- macOS рекомендуется для Apple Vision OCR.
- Linux работает для парсинга сайтов, обработки markdown, Graphify и операций с vault, но Apple Vision OCR пропускается.
- Node.js 20+.
- Python 3.11+.
- Git.
- Опционально: Obsidian, Claude Code, Codex, Cursor, OpenCode, Gemini CLI.

Установщик проверяет, что уже есть, и ставит то, что можно безопасно поставить локально. Он не требует приватных учётных данных.

## Запуск одной командой

Положите файлы в:

```text
~/Desktop/Mnemazine/inbox
```

Затем запустите:

```bash
node scripts/mnemazine-run.mjs
```

Прогон делает:

1. перепись файлов;
2. SHA-256 дедупликацию;
3. локальное извлечение текста, когда возможно;
4. создание финальной заметки с `source_ref` и `source_hash`;
5. gate качества vault;
6. перенос исходников в архив только после зелёного gate;
7. guarded-попытка обновить Graphify;
8. обновление weekly HTML-отчёта;
9. report quality gate для обновлённого weekly HTML;
10. action brief в `.mnemazine/state/last-action-brief.md`;
11. визуальный post-run отчёт по знаниям в `reports/`.

Раннер по умолчанию намеренно консервативен. Он не публикует данные, не использует приватные cookie и не отправляет локальные файлы во внешние сервисы. Он пишет локальные заметки и архивирует обработанные исходники под `.mnemazine/archive/`.

## Парсинг сайтов

Mnemazine умеет забирать сайт и превращать его полезные страницы в структурированные заметки:

```bash
node scripts/mnemazine-ingest-site.mjs --url https://example.com --apply --graphify --max-pages 40
```

Парсер ищет:

- подсказки sitemap в `robots.txt`;
- `sitemap.xml`;
- ссылки того же домена;
- заголовки страниц, описания, подзаголовки и основной текст;
- блоки JSON-LD;
- подсказки публичных API внутри same-origin JavaScript;
- ссылки на GitHub и на документацию.

По умолчанию он не использует приватные cookie или сессии браузера. Если сайту нужна авторизация — выгрузите данные сами и положите их в `inbox/`.

## Парсинг YouTube

Mnemazine умеет забирать YouTube-канал и превращать каждое видео в заметку-транскрипт, а затем автоматически подтягивать новые загрузки:

```bash
python3 scripts/kb-yt-harvest.py "https://www.youtube.com/@SomeChannel" --all --subscribe
```

Сначала он берёт субтитры (почти нулевая стоимость) и откатывается на локальный whisper, когда у видео нет пригодных субтитров. Каждое видео становится одной заметкой в inbox с именем `yt_<date>_<id>_<title>.md`. Подписанный канал затем опрашивается `scripts/kb-yt-watch.py` по RSS, забирая только новые загрузки — опционально по ежедневному расписанию launchd.

Он забирает только публичные видео и по умолчанию не использует cookie или сессии аккаунта. См. [Парсинг YouTube](docs/youtube-ingestion.ru.md).

## Контракт качества знаний

Vault должен содержать финальное знание, а не сырьё.

Каждая готовая заметка должна отвечать на вопросы:

- Что это?
- Почему это важно?
- Как это использовать?
- Где ссылки на источники?
- Что проверено, что предполагается, а что пока неизвестно?
- Какие skills, агенты, скрипты или проекты могут это переиспользовать?

Сырой OCR, скопированные фрагменты и шумные транскрипты отклоняются gate качества.

Gate качества также отклоняет типичный мусор сырого интейка: `intake-draft`, `draft-local`, `temp_image_*`, `IMG_*.PNG` и видимые расширения сырых картинок в тексте заметки. Папки вывода Graphify и бэкапы исключены из этой проверки качества заметок.

Запустить gate вручную:

```bash
node scripts/mnemazine-vault-quality-gate.mjs
```

## Отчёт знаний после прохода

После каждого полного прохода можно получить лёгкий визуальный отчёт в Markdown и HTML:

```bash
npm run postrun
```

Это не сырой лог, а справка по базе:

- какие полезные знания реально появились;
- как заметки распались на кластеры и малые атомы;
- какие дубли были учтены без мусорных заметок;
- какие 20 действий стоит взять первыми.

Если не передать JSON или логи конкретного прогона, отчёт читает свежие заметки из vault. Для точного отчёта по pipeline используйте `--results-json` или `--logs`.

## Навыки агентов

Репозиторий включает переносимые Agent Skills в стиле `.agents/skills`:

- `skills/mnemazine` — основной skill переработки знаний;
- `skills/local-doc-ops` — локальные помощники для документов/PDF.

Установщик может скопировать их в типовые места агентов, когда эти инструменты есть:

- `~/.codex/skills`;
- `~/.claude/skills`;
- проектный `.agents/skills`.

Skills безопасны для публикации: без персональных путей, приватных репозиториев, имён аккаунтов и секретов.

## Одинаковая работа Claude и Codex

Mnemazine устроен так, чтобы Claude Code и Codex выполняли один и тот же контракт знаний:

- одни и те же публичные скрипты в `scripts/`;
- одни и те же описания ролей в `agents/kb-pipeline/`;
- одинаковая дисциплина `source_ref` / `source_hash`;
- одинаковый quality gate до архивации;
- одинаковый визуальный отчёт после полного прохода.

Личности агентов являются частью workflow, а не украшением. Они описаны как публично безопасные паспорта ролей, чтобы Claude и Codex сохраняли одинаковые обязанности и стиль без приватных данных.

## Graphify

Graphify превращает vault в навигируемый граф связей. Mnemazine использует его для:

- поиска связанных заметок;
- извлечения с опорой на граф;
- еженедельных карт изменений;
- поиска дублей и почти-дублей идей;
- показа того, как один источник влияет на несколько областей знаний.

Для безопасных локальных обновлений используйте:

```bash
export MNEMAZINE_VAULT="/path/to/your/vault"
npm run graph:refresh -- --vault "$MNEMAZINE_VAULT" --mode auto
```

Эта обёртка держит `graph.json`, `GRAPH_REPORT.md`, backup/restore и `needs_update` в согласованном состоянии вместо слепого доверия одному тяжёлому семантическому прогону.

Для локальных семантических обновлений через Ollama она также использует защищённую лестницу моделей, отбраковывая модели, которые проваливают мини-смоук `graphify extract`, прежде чем они коснутся реального графа vault. API-backend тоже поддержан через переменные окружения вроде `OPENAI_API_KEY`, `ANTHROPIC_API_KEY` и `GEMINI_API_KEY`.

Дефолты репозитория лежат в `config/graphify-refresh.json`. Переопределяйте их флагами CLI или env-переменными `MNEMAZINE_GRAPHIFY_*` по необходимости.

## Еженедельный HTML-брифинг

Еженедельный отчёт — это локальная HTML-презентация, по умолчанию на русском. Он сделан так, чтобы его было приятно читать, а не как сырой лог.

Каждую карточку можно пометить локально:

- `read` — оставить в vault;
- `work` — перенести в бэклог действий;
- `forget` — убрать или поместить в карантин из активного vault.

Состояние хранится в:

```text
~/Desktop/Mnemazine/.mnemazine/state/weekly-state.json
```

## Философия репозитория

Mnemazine — это не «второй мозг» как лозунг про хранилище. Это система памяти как конвейер:

```text
сырьё -> извлечение -> понимание -> исследование -> проверка -> атомизация -> vault -> граф -> переиспользование
```

<p align="center">
  <img src="docs/assets/hero/mnemazine-pipeline.png" width="820" alt="Конвейер Mnemazine: сырьё, извлечение, проверка, синтез, vault, граф, переиспользование">
</p>

Это важно, потому что настоящая память должна быть воспроизводимой. Заметка должна уметь стать:

- skill;
- инструкцией агента;
- скриптом;
- продуктовым решением;
- чеклистом;
- еженедельным действием;
- будущим промптом с гораздо меньшим контекстом.

## Безопасность

Этот публичный репозиторий намеренно исключает:

- содержимое приватного vault;
- скриншоты с персональными данными;
- API-ключи;
- SSH-хосты;
- имена аккаунтов;
- имена приватных проектов;
- локальные абсолютные пути с исходной машины разработки.

Перед публикацией релиза запускайте полный локальный gate:

```bash
npm run release-check
```

Он проверяет синтаксис скриптов, гоняет временный demo-интейк, проверяет архивирование-после-качества, запускает gate качества demo-vault, сканирует приватные маркеры/секреты и проверяет, что публичное описание и оба README остаются двуязычными.
