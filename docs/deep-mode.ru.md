# Deep-режим (атомизация + проверка)

У Mnemazine два режима работы:

- **Консервативный (по умолчанию):** только локально. Без сети, без LLM, без внешних сервисов. Так работают `node scripts/mnemazine-run.mjs` и `npm run synthesize` по умолчанию.
- **Deep (по запросу):** использует LLM-агента (Claude по умолчанию, Codex в паритете), чтобы разбить один источник на много сфокусированных заметок (README «один источник → ~20 заметок») и проверить утверждения по их источникам.

Deep-режим **выключен, пока вы его не попросите**. Ничто в стандартном конвейере не выходит в сеть и не зовёт LLM.

## Включение deep-режима

```bash
# рабочий запуск Desktop Inbox, строгий полный протокол:
npm run protocol:desktop

# весь прогон, deep:
node scripts/mnemazine-run.mjs --deep
# или через env (форвардится в synthesize):
MNEMAZINE_DEEP=1 node scripts/mnemazine-run.mjs

# только синтез, deep:
npm run synthesize -- --deep
```

Для рабочего Desktop Inbox используй `npm run protocol:desktop`. Он сам читает `.mnemazine/config.local.sh`, берёт inbox с рабочего стола, пишет в `~/Мозг`, включает `--deep`, требует атомизацию + обогащение и после этого запускает `complete-check`.

Если deep-режим запрошен напрямую, но движок LLM недоступен, обычный `node scripts/mnemazine-run.mjs --deep` откатывается на локальный шаблонный синтез и сообщает `degraded: true` в JSON-выводе. Строгий запуск (`--require-deep` или `npm run protocol:desktop`) в такой ситуации падает до архива.

## Мост LLM

Все вызовы LLM идут через один провайдер-абстрактный модуль: `scripts/mnemazine-llm.mjs` (`llmJson(prompt, schema, {provider, tools})`). **Движок в первую очередь — Claude** (headless `claude -p`); **Codex держится в паритете** — тот же контракт, поэтому всё, что работает через Claude, работает и через Codex. Третьего LLM-клиента нет. `mnemazine-codex.mjs` остаётся тонким shim'ом обратной совместимости.

### Переменные окружения

| Переменная | По умолчанию | Назначение |
|----------|---------|---------|
| `MNEMAZINE_LLM` | `claude` | Движок: `claude` (primary) или `codex` (паритет). |
| `MNEMAZINE_CLAUDE_BIN` | авто | Claude CLI: авто-поиск через login-shell PATH, типовые установки (npm/Homebrew/standalone/Desktop), VSCode. Переопределите чтобы зафиксировать. |
| `MNEMAZINE_CODEX_BIN` | `/Applications/Codex.app/Contents/Resources/codex` | Путь к бинарю Codex. |
| `MNEMAZINE_LLM_TIMEOUT_MS` | `420000` | Таймаут одного вызова. |
| `MNEMAZINE_DEEP` | не задано | `1` включает deep-режим (enrich + атомизация + проверка + digest). |
| `MNEMAZINE_ENRICH` | `1` в deep | `0` (или `--no-enrich`) пропускает стадию обогащения. |
| `MNEMAZINE_MAX_ATOMS` | `20` | Лимит атомов на один кластер источника. |
| `MNEMAZINE_CONCURRENCY` | `4` | Размер роя для deep-исследования (параллельные агенты, ограниченно). |
| `MNEMAZINE_OWNER_CONTEXT` | общий | Контекст личных проектов для блока «почему важно» в справке. Либо положите в gitignored `.mnemazine/owner-context.txt`. |

### Распознавание (локально сперва, LLM-fallback)

Извлечение сначала пробует локальные движки за **0 токенов**: Apple Vision OCR (картинки), markitdown (PDF/DOCX/PPTX/XLSX/HTML), whisper + OCR кадров (видео). Только если локально ничего пригодного **и** включён `--deep`, файл распознаёт vision-LLM. Каждый файл изолирован — ошибка распознавания логируется, файл остаётся в inbox, остальной батч не ломается.

### Рой

В deep-режиме файлы/кластеры исследуются параллельно ограниченным пулом агентов (`MNEMAZINE_CONCURRENCY`, по умолчанию 4) — дёшево и быстро. Упавшая задача не блокирует остальные.

### Обогащение (расширение знаний)

Перед атомизацией deep-режим запускает web-агента LLM, который **исследует и расширяет** захваченный материал «насколько реально полезно» — первоисточники, актуальные факты/версии, опыт практиков — каждый добавленный факт привязан к fetched-URL (антигаллюцинация). Атомы строятся из **расширенного** знания, не только из сырого захвата. Отключение: `--no-enrich` / `MNEMAZINE_ENRICH=0`.

### Справка (русская человеческая сводка)

После Graphify `scripts/mnemazine-digest.mjs` (`npm run digest`) пишет в каждую заметку русскую **Справку** в стиле humanizer — *Что это / О чём / Почему важно мне / Связи* — плюс одну ноту-сводку прогона с картой всех атомов. Связи берутся прямо из метаданных заметок: атомы с общим `cluster_id` (родня из одного источника) и атомы с общим хостом source-URL. Детерминированно, без ключа модели. (`graphify update` строит только внутридокументный структурный код-граф; более богатые семантические связи нота↔нота потребовали бы отдельного прохода `graphify --update` и здесь не используются.) Это поверхность переиспользования: открыл одну заметку — понял знание и как оно связано. Идемпотентно (пропускает заметки со Справкой, если не `--force`).

### Атомизация (G4)

`scripts/mnemazine-synthesize.mjs` (с `--deep`/`--atomize`) отправляет каждый кластер источника в LLM и просит сфокусированные атомы — у каждого заголовок, что/зачем, пункты «как применить», следующее действие и поддерживающие URL источников. Каждый атом становится отдельной заметкой. Имена файлов — content-fingerprint (с привязкой к id кластера), поэтому повторные прогоны идемпотентны и не затирают существующие заметки.

### Проверка (G5)

`scripts/mnemazine-verify.mjs` присваивает каждой заметке `verification_status`:

- `unknown` — нет URL источника, подтверждающего утверждение;
- `assumed` — URL источника есть, но не запрашивался/не проверялся (**локальный** вердикт по умолчанию, без сети);
- `verified` — только при `--deep`: источник был достижим (HEAD/GET) **и** web-сверка LLM признала, что он подтверждает утверждение. Такие заметки получают `verified: true` и `status: final`.

## Безопасность

### Недоверенный ввод изолируется

Извлечённый материал (OCR, транскрипты, текст с сайтов) — **недоверенный**. Перед попаданием в любой промпт LLM он оборачивается `fenceUntrusted()` — делимитер инертных данных плюс явная инструкция, что содержимое нельзя исполнять как команды. Любое буквальное вхождение sentinel-метки внутри контента нейтрализуется. Это основная защита от prompt-injection через захваченный материал.

### Песочница

Backend Claude запускает `claude -p` без `--dangerously-skip-permissions`; неразрешённые инструменты просто не выполняются. Backend Codex идёт headless с bypass-флагом конвейера `kb-*-codex.sh` (неинтерактивный запуск требует). В обоих случаях активная митигация — изоляция на уровне промпта выше, а дальнейшее ужесточение песочницы backend-а — отдельное осознанное решение, не требуется для консервативного режима по умолчанию, который Codex вообще не зовёт.

### Граница данных

Deep-проверка (`--deep`) отправляет текст утверждения и URL источников в LLM-агента, который делает web-поиск — поэтому локально полученный текст достигает внешних поисковых сервисов **только при `--deep`**. Консервативный режим по умолчанию этого не делает.

### Локальный скан секретов

`npm run release-check` (и `npm run public-check`) сканируют не только то, что может уйти в публику, но и локальный кэш извлечения (`.mnemazine/cache/extracted/`) на токен-подобные секреты (API-ключи, токены, приватные ключи), потому что захваченные скриншоты или PDF могут содержать креды, которые иначе утекли бы в синтезированные заметки. Захваченный секрет роняет гейт.
